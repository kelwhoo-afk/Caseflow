# AutoCase Network — Synthetic Law Firm Documents

Synthetic document pack for the CaseRouter Auto hackathon prototype.

Purpose:
- Provide realistic law-firm knowledge documents for retrieval.
- Simulate how a car-accident referral network might store intake rules, acceptance criteria, safety policies, referral consent language, and attorney capacity.
- Use as a base for Moss retrieval and a law-firm-facing demo.

Important:
- All firms, people, policies, criteria, and client scenarios are fictional.
- These are demo documents, not legal advice.
- Referral fee and consent workflows are placeholders and must be reviewed by counsel before real-world use.

Recommended retrieval indexes:
1. auto_intake_checklists
2. firm_fit_criteria
3. safe_response_policies
4. referral_consent_policies
5. attorney_capacity_rules
6. call_scripts

Core demo flows:
- Possible fit -> attorney review -> accept and book consult.
- Clear non-fit -> client consent -> referral packet sent to partner firm.
- No injury/property damage only -> decline/resource path.
