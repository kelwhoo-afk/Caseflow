# Shared Referral Protocol — AutoCase Network

## Network rule

All network firms agree that inbound car-accident leads should be routed to the firm most likely to accept and handle the matter based on subtype, geography, severity, liability complexity, language need, and capacity.

## Confidentiality and consent

Potential client intake information is confidential and may contain sensitive personal, medical, and insurance details.

Do not share a case packet with a partner firm until:
1. The recommended receiving firm is identified.
2. The client is told which firm may receive the information.
3. The client consents to sharing the case summary and contact information.
4. The consent status is logged.

## Referral workflow

1. Intake completed.
2. System identifies originating firm fit.
3. If non-fit but network-fit, system recommends receiving firm.
4. System generates referral packet.
5. System requests client consent.
6. If consent granted, packet is sent.
7. Receiving firm reviews and marks accepted, declined, consult booked, signed, or closed.
8. Referral opportunity and fee eligibility are tracked.

## Do not auto-share

If consent is denied or unavailable, mark:
- Packet status: Locked
- Referral status: Not sent
- Next action: Send resource response or ask whether the client wants more information.

## Fee tracking language

Use "potential referral opportunity", "pending eligibility", "receivable if case signs", and "owed if applicable".
Do not represent any referral fee as guaranteed.
